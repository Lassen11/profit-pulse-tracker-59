

## План: Вкладка «Клиенты» для компании «Дело Бизнеса»

Для компании **Дело Бизнеса** на главном дашборде появится новый раздел **Клиенты юр. лиц** с возможностью добавлять клиентов, фиксировать проданную услугу, плановые платежи и отмечать их как оплаченные с автоматическим зачислением суммы на нужный счёт.

### Что появится в интерфейсе

1. На главной странице, когда выбрана компания **Дело Бизнеса**, под блоком счетов добавится секция **«Клиенты»** с таблицей:
   - Юр. лицо (название организации, ИНН опционально)
   - Услуга
   - Сумма платежа
   - Дата платежа
   - Статус: «Не оплачено» / «Оплачено» (с датой и счётом)
   - Действия: ✏️ редактировать / 🗑 удалить
2. Кнопка **«Добавить клиента»** открывает диалог:
   - Название юр. лица, ИНН (необязательно), контакт (необязательно)
   - Список услуг/платежей: можно добавить несколько строк (услуга, сумма, дата платежа). Это удобно, если у клиента несколько услуг или график платежей.
3. Возле каждого платежа — чекбокс / кнопка **«Оплачено»**. По клику открывается диалог **«Зачислить платёж»**:
   - Выбор счёта (из таблицы `accounts` для компании Дело Бизнеса)
   - Дата зачисления (по умолчанию — сегодня)
   - Подтверждение суммы
   - Кнопка «Зачислить»
4. После подтверждения:
   - Платёж помечается как оплаченный (сохраняется счёт + дата + ссылка на транзакцию)
   - Создаётся транзакция типа `income` в таблице `transactions` (компания «Дело Бизнеса», категория «Продажи», `income_account` = выбранный счёт, `organization_name` = юр.лицо, `description` = название услуги)
   - Эта транзакция автоматически отображается в **доходах** дашборда и в **балансе выбранного счёта** (логика уже работает с таблицей `transactions`).
5. При снятии отметки «Оплачено» (или удалении платежа) связанная транзакция удаляется, баланс счёта возвращается.

### Технические детали

**Новые таблицы (миграция):**

- `business_clients`
  - `id uuid pk`, `user_id uuid`, `name text` (юр. лицо), `inn text null`, `contact text null`, `created_at`, `updated_at`
  - RLS: authenticated select/insert/update/delete (по аналогии с `bankrot_clients`)
- `business_client_payments`
  - `id uuid pk`, `client_id uuid` (ref `business_clients.id`, on delete cascade — через триггер/политику)
  - `service text` (услуга), `amount numeric`, `payment_date date`
  - `is_paid boolean default false`, `paid_account text null`, `paid_at date null`
  - `transaction_id uuid null` — ссылка на созданную транзакцию для отката
  - `user_id`, `created_at`, `updated_at`
  - RLS: те же политики

**Новые компоненты:**

- `src/components/BusinessClientsSection.tsx` — таблица клиентов + платежей, кнопки добавить/редактировать/удалить, чекбокс «Оплачено».
- `src/components/BusinessClientDialog.tsx` — диалог создания/редактирования клиента и его платежей (динамический список строк платежей).
- `src/components/PaymentReceiveDialog.tsx` — диалог выбора счёта при отметке «Оплачено» (использует список из `accounts` с фильтром `company = 'Дело Бизнеса'`).

**Интеграция в Dashboard:**

- В `src/pages/Dashboard.tsx` отрендерить `<BusinessClientsSection />` только когда `selectedCompany === 'Дело Бизнеса'` (после блока счетов / KPI).
- При отметке «Оплачено» вставляется запись в `transactions` с теми же полями, что и обычная сделка через `TransactionDialog`, поэтому существующий расчёт KPI и баланса счетов подхватит её автоматически без правок.
- При снятии отметки или удалении платежа — `delete` транзакции по сохранённому `transaction_id`.

**Realtime/обновления:**

- Подписка на изменения `business_clients` и `business_client_payments` для авто-рефреша таблицы (по аналогии с уже существующими каналами на дашборде).

**Доступ:**

- Раздел скрыт от роли `manager_oz` (она и так видит только дашборд без редактирования).

### Что не входит

- Импорт/экспорт клиентов в Excel (могу добавить отдельной задачей).
- График погашения / автоматическое создание следующих платежей по периоду (сейчас платежи добавляются вручную при создании клиента — это можно расширить позже).
